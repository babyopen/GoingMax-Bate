# 业务层通用回测工具（business-common-backtest.js）

> 新增日期：2026-06-24
> 拆分自：`business/zodiac/business-zodiac-backtest.js`（5 个函数 → 3 个通用方法）
> 跨项目复用：仅依赖 `Utils.SpecialCalculator.getSpecial`，可独立打包给其他项目使用

## 一、抽取背景

原 [business-zodiac-backtest.js](file:///Users/macbook/Documents/Gemini-app/Bate/1.2版本/-3-1/business/zodiac/business-zodiac-backtest.js) 包含 5 个回测函数：

| 原函数 | 替代 | 重复度 |
|---|---|---|
| `_runGenericBacktest` | `BusinessCommonBacktest.runGeneric` | 100% 复用 |
| `runSizeBacktest` | `runGeneric` + 大小 config | 95% 重复 |
| `runOddEvenBacktest` | `runGeneric` + 单双 config | 95% 重复 |
| `runWuxingBacktest` | `runGeneric` + 五行 config | 95% 重复 |
| `runColorBacktest` | `runGeneric` + 波色 config | 95% 重复 |

**4 个具体回测函数**仅在 config 上有差异：
- `categories`（分类数组）
- `extractValue`（提取特码分类）
- `fieldNames`（字段名）
- `weights`（路径 B 权重）
- `trendPredictor` + `buildSequence`（路径 A 趋势预测器）

---

## 二、API 清单

通用文件 [business-common-backtest.js](file:///Users/macbook/Documents/Gemini-app/Bate/1.2版本/-3-1/business/business-common-backtest.js) 暴露 `BusinessCommonBacktest` 全局对象，包含 3 个方法：

| # | 方法 | 用途 |
|---|---|---|
| 1 | `runGeneric(historyData, config)` | 通用回测（端到端） |
| 2 | `buildSequence(historyData, offset, options)` | 通用 sequence 构造（路径 A 辅助） |
| 3 | `summarizeResults(results, recentCount)` | 计算汇总数据（命中率 / 连中） |

---

## 三、两条预测路径

通用函数支持**两种预测路径**，由 config 自动选择：

| 路径 | 触发条件 | 说明 | 与实际推荐一致性 |
|---|---|---|---|
| **A** | 同时提供 `trendPredictor` + `buildSequence` | 调业务方 trendPredictor(sequence) 拿预测 | ✅ 100% 一致 |
| **B** | 仅提供 `extractValue` + `weights` | 通用打分（连出 / 交替 / 重复 / 惯性 / 统计） | ⚠️ 旧算法 |

原项目 4 个回测函数**全部使用路径 A**（与实际推荐一致）。

## 四、使用示例

### 示例 1：替换 `runSizeBacktest`（大小回测）

```js
// 旧调用（保留，仍可用）
const sizeData = ZodiacPrediction.runSizeBacktest(historyData, 12);

// 新调用（推荐，路径 A）
const sizeData = BusinessCommonBacktest.runGeneric(historyData, {
  testCount: 12,
  categories: ['大', '小'],
  extractValue: function(special, item) {
    return item && (item.te >= 25 ? '大' : '小');
  },
  getNumber: function(special, item) {
    return item && item.te;
  },
  fieldNames: { predicted: 'predictedSize', actual: 'actualSize' },
  trendPredictor: function(sequence) {
    return BusinessCommonStats.predictBinaryTrend(sequence, {
      fieldName: 'size',
      labelA: '大',
      labelB: '小'
    });
  },
  buildSequence: function(historyData, offset) {
    return BusinessCommonBacktest.buildSequence(historyData, offset, {
      windowSize: 10,
      entryBuilder: function(item) {
        return { expect: item.expect, number: item.te, size: item.te >= 25 ? '大' : '小' };
      }
    });
  }
});
```

### 示例 4：路径 B（通用打分）— 不使用 trendPredictor

```js
// 路径 B：纯通用打分（连出 / 交替 / 重复 / 惯性 / 统计）
const result = BusinessCommonBacktest.runGeneric(historyData, {
  testCount: 12,
  categories: ['大', '小'],
  extractValue: function(special, item) {
    return item && (item.te >= 25 ? '大' : '小');
  },
  getNumber: function(special, item) { return item && item.te; },
  fieldNames: { predicted: 'predictedValue', actual: 'actualValue' },
  weights: {
    consecutive: 35,   // 连续3期同号
    alternate: 25,     // 3期交替
    repeat: 15,        // 第 3 期重复
    inertia: 10,       // 最近 2 期同号
    statistical: 12    // 占比 40-60% 略占优（仅二分类）
  },
  maxConfidence: 72,
  baseConfidence: 42,
  confidenceRange: 28,
  fallbackConfidence: 40
});
```

## 五、返回值字段

`runGeneric` / `summarizeResults` 返回值：

| 字段 | 类型 | 说明 |
|---|---|---|
| `totalTests` | Number | 总回测期数 |
| `totalHits` | Number | 总命中次数 |
| `totalHitRate` | Number | 总命中率（%，0-100） |
| `recentTests` | Number | 最近 N 期回测数 |
| `recentHits` | Number | 最近 N 期命中数 |
| `recentHitRate` | Number | 最近 N 期命中率（%） |
| `currentStreak` | Number | 当前连中数 |
| `details` | Array | 最近 N 期明细，每项含 `expect` / `actualNumber` / `confidence` / `isHit` / 字段名对应的 predicted/actual |

## 七、与原函数的差异

按"只能新增"原则，**原函数保留不变**。通用函数与原函数：

| 维度 | 状态 |
|---|---|
| 算法逻辑 | ✅ 100% 一致 |
| 返回字段 | ✅ 100% 一致 |
| 默认参数 | ✅ 100% 一致（路径 A 走 trendPredictor，结果与原函数完全一致） |
| 路径 B 边界行为 | ✅ 100% 一致（保留所有 `if (config.weights.xxx)` 边界） |

**唯一差异**：`extractValue` / `getNumber` 回调签名变为 `(special, item)` 双参数（兼容原 `(item)` 单参），不影响行为。

## 八、跨项目复用

`business-common-backtest.js` **不依赖任何项目特定工具**，仅需：
- `historyData` 数组（含 `expect` 字段）
- `extractValue` / `getNumber` / `trendPredictor` / `buildSequence` 等回调由业务方提供

**跨项目使用步骤**：
1. 复制 `business-common-backtest.js` 到目标项目
2. 在 index.html 加载
3. 直接调用 `BusinessCommonBacktest.runGeneric(historyData, config)` 即可

## 十、性能

- **路径 A**：业务方 trendPredictor 仅被调用 N 次（N = testCount），效率与原函数一致
- **路径 B**：纯计算，无副作用
- **汇总计算**（summarizeResults）：O(N) 单次遍历
