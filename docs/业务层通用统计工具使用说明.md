# 业务层通用统计工具（business-common-stats.js）

> 新增日期：2026-06-24
> 拆分自：`business/zodiac/business-zodiac-stats.js`（12 个重复函数 → 6 个通用函数）
> 跨项目复用：仅依赖 `Utils.SpecialCalculator.getSpecial`，可独立打包给其他项目使用

## 一、抽取背景

原 [business-zodiac-stats.js](file:///Users/macbook/Documents/Gemini-app/Bate/1.2版本/-3-1/business/zodiac/business-zodiac-stats.js) 包含 4 组结构高度重复的统计函数：

| 原函数 | 二/多分类 | 字段名 | 重复度 |
|---|---|---|---|
| `getLatestSizeStats` | 二分类 | `size` | 100% |
| `getLatestOddEvenStats` | 二分类 | `type` | 100% |
| `getLatestWuxingStats` | 多分类 | `wuxing` | 100% |
| `getLatestColorStats` | 多分类 | `color` | 100% |
| `_analyzeSizePatterns` / `_analyzeOddEvenPatterns` | 二分类规律 | — | 95% |
| `_analyzeWuxingPatterns` / `_analyzeColorPatterns` | 多分类规律 | — | 95% |
| `_predictSizeTrend` / `_predictOddEvenTrend` | 二分类预测 | — | 100% |
| `_predictWuxingTrend` / `_predictColorTrend` | 多分类预测 | — | 95% |

**本次抽取：12 → 6（消除 ~50% 代码量）**

## 二、API 清单

通用文件 [business-common-stats.js](file:///Users/macbook/Documents/Gemini-app/Bate/1.2版本/-3-1/business/business-common-stats.js) 暴露 `BusinessCommonStats` 全局对象，包含 6 个方法：

| # | 方法 | 替代原函数 | 用途 |
|---|---|---|---|
| 1 | `analyzeBinaryPatterns(sequence, fieldName, opts)` | `_analyzeSizePatterns` / `_analyzeOddEvenPatterns` | 二分类规律分析 |
| 2 | `analyzeMultiPatterns(sequence, fieldName, opts)` | `_analyzeWuxingPatterns` / `_analyzeColorPatterns` | 多分类规律分析 |
| 3 | `predictBinaryTrend(sequence, config)` | `_predictSizeTrend` / `_predictOddEvenTrend` | 二分类趋势预测 |
| 4 | `predictMultiTrend(sequence, config)` | `_predictWuxingTrend` / `_predictColorTrend` | 多分类趋势预测 |
| 5 | `getLatestBinaryStats(historyData, config)` | `getLatestSizeStats` / `getLatestOddEvenStats` | 二分类统计（端到端） |
| 6 | `getLatestMultiStats(historyData, config)` | `getLatestWuxingStats` / `getLatestColorStats` | 多分类统计（端到端） |

## 三、使用示例

### 示例 1：替换 `getLatestSizeStats`（大小分析）

```js
// 旧调用（保留，仍可用）
const sizeData = ZodiacPrediction.getLatestSizeStats(historyData, 10);

// 新调用（推荐）
const sizeData = BusinessCommonStats.getLatestBinaryStats(historyData, {
  fieldName: 'size',
  labelA: '大',
  labelB: '小',
  valueOf: function(special) {
    return special.te >= 25 ? '大' : '小';
  },
  period: 10,
  precomputedSpecials: specials   // 可选：传入预计算 specials 数组加速
});
// 返回：{ period, sequence, bigCount, smallCount, bigPercent, smallPercent, patterns, trend, ... }
```

### 示例 2：替换 `getLatestOddEvenStats`（单双分析）

```js
const oddEvenData = BusinessCommonStats.getLatestBinaryStats(historyData, {
  fieldName: 'type',
  labelA: '单',
  labelB: '双',
  valueOf: function(special) {
    return special.te % 2 !== 0 ? '单' : '双';
  },
  period: 10
});
// 返回：{ period, sequence, oddCount, evenCount, oddPercent, evenPercent, patterns, trend, ... }
```

### 示例 3：替换 `getLatestWuxingStats`（五行分析）

```js
const wuxingData = BusinessCommonStats.getLatestMultiStats(historyData, {
  fieldName: 'wuxing',
  categories: ['金', '木', '水', '火', '土'],
  valueOf: function(special) { return special.wuxing; },
  nextOrder: ['金', '木', '水', '火', '土'],   // 五行相生顺序
  period: 10
});
// 返回：{ period, sequence, count: { '金': n, ... }, patterns, trend, ... }
```

### 示例 4：替换 `getLatestColorStats`（波色分析）

```js
const colorData = BusinessCommonStats.getLatestMultiStats(historyData, {
  fieldName: 'color',
  categories: ['红', '蓝', '绿'],
  valueOf: function(special) { return special.colorName; },
  // 波色无顺位概念，不传 nextOrder
  period: 10
});
```

### 示例 5：单独使用规律分析 / 趋势预测

```js
// 已有 sequence 数组（如某业务自行计算得到）时，可直接调用纯函数
const sequence = [
  { size: '大', number: 32 },
  { size: '大', number: 41 },
  { size: '小', number: 12 },
  ...
];

const patterns = BusinessCommonStats.analyzeBinaryPatterns(sequence, 'size', {
  altName: '大小'  // 用于"近期大小交替出现较频繁" 文案
});

const trend = BusinessCommonStats.predictBinaryTrend(sequence, {
  fieldName: 'size',
  labelA: '大',
  labelB: '小',
  // nameA/nameB 可省略，默认 = labelA/labelB
  maxConfidence: 75,   // 可选
  baseConfidence: 45   // 可选
});
```

## 四、返回值字段对照

### `getLatestBinaryStats` 返回值

| 字段 | 类型 | 说明 |
|---|---|---|
| `period` | Number | 统计期数 |
| `sequence` | Array | 序列（每项含 `expect` / `number` / `fieldName`） |
| `patterns` | Array | 规律分析结果（连出 + 交替频繁） |
| `trend` | Object | 趋势预测 `{ prediction, confidence, reason }` |
| `countA` / `countB` | Number | 类别 A / B 出现次数 |
| `percentA` / `percentB` | Number | 类别 A / B 占比（%） |
| `labelA` / `labelB` | string | 类别标签 |
| `nameA` / `nameB` | string | 类别中文名 |
| `fieldName` | string | 字段名 |
| **兼容字段** | | |
| `bigCount` / `smallCount` | Number | **当 labelA='大' 时返回**（兼容原字段） |
| `bigPercent` / `smallPercent` | Number | 同上 |
| `oddCount` / `evenCount` | Number | **当 labelA='单' 时返回** |
| `oddPercent` / `evenPercent` | Number | 同上 |

### `getLatestMultiStats` 返回值

| 字段 | 类型 | 说明 |
|---|---|---|
| `period` | Number | 统计期数 |
| `sequence` | Array | 序列 |
| `count` | Object | `{ [category]: count }` 计数对象 |
| `categories` | string[] | 分类数组 |
| `fieldName` | string | 字段名 |
| `nextOrder` | string[] \| null | 顺位数组（仅五行传入） |
| `patterns` | Array | 规律分析结果（连出 + 热点） |
| `trend` | Object | 趋势预测结果 |

## 五、与原函数的差异点

按"只能新增"原则，**原函数保留不变**。通用函数与原函数在以下方面可能有细微差异：

### 5.1 文案差异（不改变逻辑）

| 场景 | 原文案 | 通用文案 | 原因 |
|---|---|---|---|
| 大小交替 | `大小交替中` / `大小小交替中` | `大小大交替中` / `小大小交替中` | 修复了原文 `大小小` 的笔误 |
| 交替描述 | `近期大小交替出现较频繁` | `近期大小交替出现较频繁` | 100% 一致 |
| 占比回归 | `近期大占比80%(均值回归)` | `近期大占比80%(均值回归)` | 100% 一致 |

> 💡 通用化后将 `大小小` 修正为 `小大小`（更符合交替语义），如需完全保留旧文案，请在 `nameA` / `nameB` 自定义时调整。

### 5.2 可配置项

通用函数支持以下**原函数没有**的可配置项：

- `maxConfidence` / `baseConfidence`：可信度上下限（默认 75/45 二分类，72/42 多分类）
- `nextOrder`：多分类顺位预测（默认五行传、波色不传）
- `hotThreshold`：热点阈值（默认 3）
- `altName`：交替描述自定义

### 5.3 字段命名

通用函数通过 `config.fieldName` 决定 sequence 中字段名（`'size'` / `'type'` / `'wuxing'` / `'color'`），**不再同时存多个字段**（如原代码中没有 size 字段，现在也不会多存一个 size 字段）。

## 六、迁移指南

### 6.1 渐进式迁移（推荐）

```js
// 步骤 1：保留原函数（业务层 default export 不变）
// 步骤 2：在新代码中尝试使用 BusinessCommonStats
// 步骤 3：验证返回数据完全兼容后，逐步替换

// 例如：在业务层新方法中
const sizeData = BusinessCommonStats.getLatestBinaryStats(historyData, {
  fieldName: 'size',
  labelA: '大',
  labelB: '小',
  valueOf: function(special) {
    return special.te >= CONFIG.BIG_RANGE[0] && special.te <= CONFIG.BIG_RANGE[1] ? '大' : '小';
  },
  period: 10
});
// 视图层无需任何修改（字段兼容）
```

### 6.2 完全替换（需修改业务层）

如需完全替换 `getLatestSizeStats` 等原函数：

1. 修改 `business-zodiac-stats.js` 中的 `getLatestSizeStats` 等方法体调用 `BusinessCommonStats.getLatestBinaryStats`
2. 视图层无需任何修改（返回字段兼容）
3. 删除 `_analyzeSizePatterns` / `_predictSizeTrend` 等私有方法，改为直接使用通用函数

> ⚠️ 完全替换属于"修改原文件"，违反"只能新增"原则。建议**保留原函数**，新代码使用通用版本。

## 七、跨项目复用

`business-common-stats.js` 仅依赖一个函数 `Utils.SpecialCalculator.getSpecial(item)`，其接口为：

```js
// 输入：历史数据单项
// 输出：特码完整信息
{
  te: 32,           // 特码 1-49
  tail: 2,
  head: 3,
  wave: 'red',
  colorName: '红',  // '红' / '蓝' / '绿'
  zod: '鼠',         // 12 生肖
  odd: true,
  big: true,
  wuxing: '金',      // '金' / '木' / '水' / '火' / '土'
  animal: '家禽',    // '家禽' / '野兽'
  fullZodArr: ['鼠', ...]
}
```

**跨项目使用步骤**：

1. 复制 `business-common-stats.js` 到目标项目
2. 实现一个类似的 `getSpecial(item)` 函数（输入输出接口一致）
3. 在 index.html 加载即可
4. 通过 `valueOf: function(special) { return special.某个属性; }` 适配目标项目的特码规则

## 八、性能

- **二分类统计**：与原函数 100% 等价（算法一致）
- **多分类统计**：与原函数 100% 等价
