## 二、滚动卡顿修复（P0 致命级，5 处）

### 2.1 移除 `will-change: scroll-position` 长期挂载

**文件**：[css/global.css](file:///Users/macbook/Documents/Gemini-app/Bate/1.2版本/-3-1/css/global.css)

**问题**：`.page-scroll` 长期挂着 `will-change: scroll-position`，浏览器持续为该元素分配独立合成层 + GPU 内存，低端手机滚动卡成 PPT。

**修复**：删除 `will-change: scroll-position`。`will-change` 只能临时使用（动画前加，结束后移除），不能长期挂载。

### 2.2 移除双重合成层（`translateZ(0)` + `will-change`）

**文件**：[css/global.css](file:///Users/macbook/Documents/Gemini-app/Bate/1.2版本/-3-1/css/global.css)

**问题**：`translateZ(0)` 已经强制创建合成层，再加 `will-change` 是冗余，双重合成层白白多占 GPU 内存。

**修复**：同时删除 `-webkit-transform: translateZ(0)` 和 `transform: translateZ(0)`。

### 2.3 滚动监听挂错对象 + 返回顶部按钮无效

**文件**：
- [event.js](file:///Users/macbook/Documents/Gemini-app/Bate/1.2版本/-3-1/event.js) — `window.addEventListener('scroll', ...)` → `.page-scroll.addEventListener('scroll', ...)`
- [views/view-filter.js](file:///Users/macbook/Documents/Gemini-app/Bate/1.2版本/-3-1/views/view-filter.js) — `backToTop` / `getScrollTop` / `cleanupPageEvents` 改用 `.page-scroll`

**问题**：`html { overflow:hidden }` + `body { overflow:hidden; position:fixed }`，真实滚动发生在 `.page-scroll` 上，但：
- `handleScroll` 监听 `window.scroll` → **永远不触发**（导航栏不会自动隐藏）
- `backToTop` 调用 `window.scrollTo()` → **永远不会执行**（返回顶部按钮无效）
- `getScrollTop` 读 `document.documentElement.scrollTop` → **永远返回 0**

**修复**：全部改挂到 `.page-scroll` 元素上。

### 2.4 Swiper 容器 `touch-action` 去掉 `pan-y`（频率 Swiper）

**文件**：[css/layout.css](file:///Users/macbook/Documents/Gemini-app/Bate/1.2版本/-3-1/css/layout.css)

**问题**：`.freq-swiper-container` 和 `.freq-swiper-wrapper` 设置了 `touch-action: pan-x pan-y pinch-zoom`，当用户手指在 swiper 卡片上做垂直滑动时，事件被 swiper 拦截，外层 `.page-scroll` 收不到滚动事件。

**修复**：改为 `touch-action: pan-x pinch-zoom`（只声明水平滚动 + 缩放），垂直滚动穿透到外层。同时移除冗余的 `-webkit-overflow-scrolling: touch` 和 `transform: translateZ(0)`。

### 2.5 区域 Swiper 容器同样去掉 `pan-y`

**文件**：[css/layout.css](file:///Users/macbook/Documents/Gemini-app/Bate/1.2版本/-3-1/css/layout.css)

**问题**：同 2.4，`.zone-swiper-container` 和 `.zone-swiper-wrapper` 同样拦截垂直滚动。

**修复**：同上，改为 `touch-action: pan-x pinch-zoom`。

## 三、性能优化（P1，3 处）

### 3.1 倒计时只在数字变化时更新

**文件**：[business/business-main.js](file:///Users/macbook/Documents/Gemini-app/Bate/1.2版本/-3-1/business/business-main.js)

**问题**：`startCountdown` 每秒调用 `ViewAnalysis.updateCountdown()` 写入 DOM，但 60 秒里只有 1 次文本实际变化，其余 59 次是重复写入，与滚动争抢主线程。

**修复**：缓存上次文本 `_lastCountdownText`，文本不变时跳过 DOM 更新。

### 3.2 资料页回测计算改用 `requestIdleCallback`

**文件**：[business/business-main.js](file:///Users/macbook/Documents/Gemini-app/Bate/1.2版本/-3-1/business/business-main.js)

**问题**：`initGiongTab` / `initUltimateAlgorithm` / `initZodiacBacktest` 中用 `setTimeout(100/150ms)` 推迟回测渲染，但 setTimeout 在延迟到达后立即执行，如果用户正在滚动会被打断。

**修复**：3 处改为 `requestIdleCallback`（含 polyfill，Safari < 16.4 回退 setTimeout），浏览器空闲时才执行回测计算，不打扰滚动。

### 3.3 移除 `-webkit-overflow-scrolling: touch`

**文件**：[css/global.css](file:///Users/macbook/Documents/Gemini-app/Bate/1.2版本/-3-1/css/global.css)

**问题**：iOS 14+ 已废弃 `-webkit-overflow-scrolling: touch`，现代 Safari 使用原生 momentum scrolling，该属性在 iOS 14+ 上反而可能卡顿。

**修复**：删除该属性。
