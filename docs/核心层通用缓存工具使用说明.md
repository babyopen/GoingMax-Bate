## 三、API 详细说明

### 3.1 `createLRU(maxSize)`

创建 LRU 缓存实例，限制最大数量。Map 实现，O(1) 读写。

```js
const cache = CommonCache.createLRU(50);
cache.set('key1', 'value1');
cache.get('key1');  // 'value1'
cache.has('key1');  // true
cache.size;          // 1
cache.clear();
```

### 3.2 `memoize(fn, keyFn?, ttl?)`

函数记忆化，自动缓存函数返回值。**注意**：无限增长，长期运行会内存泄漏。

```js
const computeExpensive = function(n) {
  // 复杂计算...
  return n * n;
};
const cached = CommonCache.memoize(computeExpensive);
cached(5);  // 计算
cached(5);  // 命中缓存

// 带 TTL：5 秒后失效
const cachedTTL = CommonCache.memoize(fetchData, null, 5000);
```

### 3.3 `memoizeLRU(fn, maxSize?, keyFn?)`

**推荐使用**。LRU + 记忆化，带容量上限，避免内存无限增长。

```js
const _calc = CommonCache.memoizeLRU(function(historyData) {
  // 重型计算
  return computeResult(historyData);
}, 500);  // 上限 500 条

_calc(data);  // 计算
_calc(data);  // 命中缓存
```

### 3.4 `throttle(fn, delay)`

节流：高频事件优化（scroll / resize），**首次触发立即执行**。

```js
const handleScroll = CommonCache.throttle(function() {
  console.log('scroll');
}, 100);
window.addEventListener('scroll', handleScroll);
```

### 3.5 `debounce(fn, delay)`

防抖：高频点击优化，**停止触发 delay ms 后才执行**。

```js
const handleSearch = CommonCache.debounce(function(query) {
  console.log('search:', query);
}, 300);
input.addEventListener('input', function(e) {
  handleSearch(e.target.value);
});
```

### 3.6 `delayedToggle(fn, delay?)`

防反弹的延迟单次执行器。`trigger()` 排队 50ms 后执行，期间可 `cancel()`。

```js
const handle = CommonCache.delayedToggle(function() {
  toggleQuickNav();
}, 50);

btn.addEventListener('click', function() {
  handle.trigger();  // 排队
});

// 主动取消（如点击 nav-tab 关闭 quickNav）
navTab.addEventListener('click', function() {
  handle.cancel();
});
```

### 3.7 `TimerManager`

命名空间对象，按名称管理 `setTimeout` / `setInterval`，避免内存泄漏。

```js
// 启动定时器
CommonCache.TimerManager.setInterval('countdown', function() {
  updateCountdown();
}, 1000);

// 清除指定定时器
CommonCache.TimerManager.clearInterval('countdown');

// 一次性定时器
CommonCache.TimerManager.setTimeout('initDelay', function() {
  doInit();
}, 500);

// 页面卸载时统一清除
CommonCache.TimerManager.clearAll();

// 调试：查看当前活跃定时器数
const stats = CommonCache.TimerManager.getStats();
// { timeouts: 1, intervals: 3 }
```

## 四、与 Utils.* 的关系

| Utils.* | CommonCache.* | 关系 |
|---|---|---|
| `Utils.createLRU` | `CommonCache.createLRU` | 100% 等价，CommonCache 是独立实现 |
| `Utils.memoize` | `CommonCache.memoize` | 100% 等价 |
| `Utils.memoizeLRU` | `CommonCache.memoizeLRU` | 100% 等价 |
| `Utils.throttle` | `CommonCache.throttle` | 100% 等价 |
| `Utils.debounce` | `CommonCache.debounce` | 100% 等价 |
| `Utils.delayedToggle` | `CommonCache.delayedToggle` | 100% 等价 |
| `Utils.TimerManager` | `CommonCache.TimerManager` | 100% 等价 |

**调用方无需任何修改**：
- 旧调用 `Utils.throttle(fn, 100)` → 仍然可用
- 新调用 `CommonCache.throttle(fn, 100)` → 推荐使用
- 两个命名空间**互相独立**（不共享内部状态），新代码应统一改用 `CommonCache.*`

## 七、性能与最佳实践

| 场景 | 推荐 API |
|---|---|
| 单次计算结果缓存 | `memoize`（带 TTL） |
| 高频调用 + 长期运行 | `memoizeLRU`（带容量上限） |
| scroll / resize 事件 | `throttle` |
| 搜索框 input | `debounce` |
| 多次点击防反弹 | `delayedToggle` |
| 倒计时 / 心跳 | `TimerManager.setInterval` |
| 一次性延迟操作 | `TimerManager.setTimeout` |

> ⚠️ `memoize` 无限增长，长期运行可能内存泄漏。**优先使用 `memoizeLRU`**。
