# 事件总线（BusinessEventBus）使用说明

## 📖 概述

`BusinessEventBus` 是一个轻量级的事件总线模块，基于观察者模式实现，用于解耦项目中各层之间的直接依赖。

**位置**: `business/business-event-bus.js`  
**版本**: v1.0.0 (2026-06-30)

---

## 🎯 设计目标

1. **解耦核心层与平台层**: 避免 `core/state.js` 直接调用 `Render.renderAll()`
2. **提高可测试性**: 事件流可以独立测试
3. **增强可扩展性**: 新功能只需订阅/发布事件，无需修改现有代码
4. **遵循架构宪法**: 符合"单向依赖"原则

---

## 🚀 快速开始

### 基本用法

```javascript
// 1. 订阅事件
const off = BusinessEventBus.on('data:update', function(newData) {
  console.log('数据已更新:', newData);
});

// 2. 发布事件
BusinessEventBus.emit('data:update', { count: 42 });

// 3. 取消订阅
off();
```

### 一次性订阅

```javascript
// 只在第一次触发时执行，之后自动取消订阅
BusinessEventBus.once('init:complete', function() {
  console.log('应用初始化完成');
});
```

### 清除所有监听器

```javascript
// 清除特定事件的监听器
BusinessEventBus.clear('data:update');

// 清除所有事件的监听器
BusinessEventBus.clear();
```

## 📚 API 参考

### on(event, callback)

订阅事件。

**参数**:
- `event` (string): 事件名称
- `callback` (Function): 回调函数

**返回**: 
- Function: 取消订阅函数

**示例**:
```javascript
const off = BusinessEventBus.on('user:login', function(user) {
  console.log('用户登录:', user.name);
});

// 稍后取消订阅
off();
```

---

### emit(event, ...args)

发布事件。

**参数**:
- `event` (string): 事件名称
- `...args` (any): 传递给回调的参数

**示例**:
```javascript
// 发布事件，传递多个参数
BusinessEventBus.emit('order:create', orderId, orderData, timestamp);
```

---

### once(event, callback)

一次性订阅（触发后自动取消）。

**参数**:
- `event` (string): 事件名称
- `callback` (Function): 回调函数

**示例**:
```javascript
BusinessEventBus.once('app:ready', function() {
  console.log('应用就绪（只执行一次）');
});
```

---

### clear([event])

清除监听器。

**参数**:
- `event` (string, 可选): 事件名称；不传则清除所有

**示例**:
```javascript
// 清除特定事件
BusinessEventBus.clear('data:update');

// 清除所有事件
BusinessEventBus.clear();
```

---

### listenerCount(event)

获取某个事件的监听器数量（调试用）。

**参数**:
- `event` (string): 事件名称

**返回**: 
- number: 监听器数量

**示例**:
```javascript
const count = BusinessEventBus.listenerCount('data:update');
console.log(`当前有 ${count} 个监听器`);
```

## 💡 使用场景

### 场景 1: 状态变更通知

```javascript
// 在 app.js 中注册渲染监听器
BusinessEventBus.on('state:change', () => {
  Render.renderAll();
});

// 在业务代码中触发状态变更
function updateData(newData) {
  StateManager.setState({ data: newData });
  BusinessEventBus.emit('state:change');
}
```

### 场景 2: 模块间通信

```javascript
// 模块 A: 发布事件
function saveFilter(filter) {
  Storage.saveFilter(filter);
  BusinessEventBus.emit('filter:saved', filter);
}

// 模块 B: 订阅事件
BusinessEventBus.on('filter:saved', function(filter) {
  Toast.show(`方案 "${filter.name}" 已保存`);
  ViewFilter.refreshList();
});
```

### 场景 3: 异步操作完成通知

```javascript
// 数据加载完成后通知
fetchHistoryData()
  .then(data => {
    StateManager.setState({ historyData: data });
    BusinessEventBus.emit('history:loaded', data.length);
  })
  .catch(err => {
    BusinessEventBus.emit('history:error', err);
  });

// 监听加载完成
BusinessEventBus.on('history:loaded', function(count) {
  Toast.show(`已加载 ${count} 期数据`);
});

BusinessEventBus.on('history:error', function(err) {
  Toast.show('数据加载失败');
});
```

## ⚠️ 注意事项

### 1. 避免内存泄漏

**错误示例**:
```javascript
// ❌ 忘记取消订阅，导致内存泄漏
BusinessEventBus.on('data:update', handler);
```

**正确示例**:
```javascript
// ✅ 在组件销毁时取消订阅
const off = BusinessEventBus.on('data:update', handler);

// 组件卸载时
componentWillUnmount() {
  off();
}
```

### 2. 事件命名规范

建议使用 `模块:动作` 的命名格式：

```javascript
// ✅ 推荐
'state:change'
'filter:saved'
'history:loaded'
'user:login'

// ❌ 不推荐
'dataUpdate'
'saveFilter'
'loadHistory'
```

### 3. 错误处理

事件总线会捕获回调中的错误，但建议自行处理：

```javascript
BusinessEventBus.on('data:update', function(data) {
  try {
    // 处理数据
    processData(data);
  } catch (e) {
    console.error('数据处理失败:', e);
    Toast.show('数据处理出错');
  }
});
```

### 4. 性能考虑

- 避免在高频事件中执行耗时操作
- 使用防抖/节流优化频繁触发的事件
- 及时取消不再需要的订阅

```javascript
// 优化高频事件
const debouncedHandler = Utils.debounce(function(data) {
  // 处理数据
}, 300);

BusinessEventBus.on('scroll:update', debouncedHandler);
```

## 🔍 调试技巧

### 查看监听器数量

```javascript
// 检查某个事件有多少监听器
console.log('state:change 监听器数量:', 
  BusinessEventBus.listenerCount('state:change'));
```

### 临时监听所有事件

```javascript
// 开发环境下可以添加全局监听器
if (process.env.NODE_ENV === 'development') {
  const originalEmit = BusinessEventBus.emit;
  BusinessEventBus.emit = function(event, ...args) {
    console.log(`[EventBus] 发布事件: ${event}`, args);
    return originalEmit.apply(this, [event, ...args]);
  };
}
```

## 📊 与原有模式的对比

### 之前：直接调用

```javascript
// core/state.js
setState(partialState) {
  this._state = { ...this._state, ...partialState };
  Render.renderAll(); // ❌ 核心层直接调用平台层
}
```

**问题**:
- 违反单向依赖原则
- 难以替换渲染逻辑
- 测试困难

### 现在：事件总线

```javascript
// core/state.js
setState(partialState) {
  this._state = { ...this._state, ...partialState };
  BusinessEventBus.emit('state:change'); // ✅ 只发布事件
}

// app.js
BusinessEventBus.on('state:change', () => {
  Render.renderAll(); // ✅ 由入口层决定如何响应
});
```

**优势**:
- 符合单向依赖
- 易于扩展（可添加多个监听器）
- 便于测试和调试

## 🧪 单元测试示例

```javascript
import { describe, it, expect } from '@jest/globals';

describe('BusinessEventBus', () => {
  it('应该能够订阅和发布事件', () => {
    const mockCallback = jest.fn();
    
    BusinessEventBus.on('test:event', mockCallback);
    BusinessEventBus.emit('test:event', 'data');
    
    expect(mockCallback).toHaveBeenCalledWith('data');
  });
  
  it('应该能够取消订阅', () => {
    const mockCallback = jest.fn();
    const off = BusinessEventBus.on('test:event', mockCallback);
    
    off();
    BusinessEventBus.emit('test:event');
    
    expect(mockCallback).not.toHaveBeenCalled();
  });
  
  it('应该支持一次性订阅', () => {
    const mockCallback = jest.fn();
    
    BusinessEventBus.once('test:once', mockCallback);
    BusinessEventBus.emit('test:once');
    BusinessEventBus.emit('test:once');
    
    expect(mockCallback).toHaveBeenCalledTimes(1);
  });
});
```
