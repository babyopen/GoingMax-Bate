## 三、待优化的方向（受"只能新增"约束，需手动决策）

### 3.1 业务层 DOM 操作（红线 6）

**位置**：[business-main.js](../business/business-main.js) L602、L603-605、L1349-1350、L1391-1394、L1871-1872

**违规示例**：
```js
// ❌ 业务层直接获取 DOM
const customNumEl = document.getElementById('customNum');
```

**建议优化方向**（用户手动决定是否启用）：
- 方案 A：保持现状（已有 `domValues` 参数支持规范调用）
- 方案 B：在事件层 `event.js` 中读取 DOM 后传入 `Business.syncAnalyze(domValues)`
- **影响**：方案 B 需修改 `event.js` 中相关调用点，违反"只能新增"

### 3.2 核心层调用 Render（红线 10）

**位置**：[core/state.js](../core/state.js) L75、L82、L119 — 调 `Render.renderAll()`

**说明**：核心层理论上应只持有状态，不应触发渲染。但 React/Vue 等框架也常用此模式。

**建议**：
- 方案 A：保持现状（观察者模式）
- 方案 B：改为发布订阅（StateManager.emit('change') → Render.subscribe）

**约束**：`core/state.js` 是只读文件，无法修改。

### 3.3 业务层调用 View（红线 10）

**位置**：[business-main.js](../business/business-main.js) 共 20+ 处 `ViewAnalysis.*` 调用

**建议**：
- 方案 A：保持现状（业务完成后直接渲染，业务层"知道"UI 存在）
- 方案 B：业务层只产出数据，由 `event.js` 监听业务完成事件后调用 View

**约束**：`business-main.js` 是核心文件，按"只新增"原则不应修改。

### 3.4 CSS hover 残留（红线 11）

**位置**：[css/card.css](../css/card.css) L1787 — `.combined-tab:hover`

**建议**：
- 直接删除该 4 行 CSS（无任何破坏性，纯样式清理）
- **但违反"只能新增"**：删除属于破坏行为

### 3.5 视图层鼠标事件（红线 12）

**位置**：[views/view-common.js](../views/view-common.js) L242-329 — 含 `mousedown / mousemove / mouseup / mouseleave` 13 处

**说明**：这是触屏滑动 swiper 组件，需要兼容鼠标拖拽（PC 端测试用）。

**建议**：
- 方案 A：保持现状（功能需要）
- 方案 B：新增纯触屏版本，原文件保留兜底

**约束**：违反"只能新增"。

### 3.6 event.js 含 DOM 操作（红线 9）

**位置**：[event.js](../event.js) 共 51 处 DOM 操作

**说明**：事件层需要处理 UI 状态切换（展开/折叠卡片、显示/隐藏 iframe loading）。

**建议**：
- 方案 A：保持现状（事件层需要根据 DOM 状态做轻量操作）
- 方案 B：把所有"展开/折叠"逻辑下沉到 `ViewZodiacGiong.togglePanel(...)` 等视图函数

**约束**：违反"只能新增"。

## 五、规则本身的可优化点

### 5.1 规则冲突点 1：业务层 ↔ 视图层

**现状**：业务层（`business-main.js`）频繁调用 `ViewAnalysis.*`，违反"单向依赖"。
**规则建议**：业务层应只产出数据 + 调用 `Render.*` 通用渲染；具体页面渲染由 `event.js` 调度。

### 5.2 规则冲突点 2：核心层 ↔ 平台层

**现状**：`core/state.js` 调 `Render.renderAll()`。
**规则建议**：核心层应通过事件总线 / 订阅模式通知上层，禁止直接调用。

### 5.3 规则冲突点 3：事件层职责

**现状**：`event.js` 含 51 处 DOM 操作，违反"禁止渲染代码"。
**规则建议**：补充"事件层可通过 data-action 委托做轻量 UI 状态切换（如 .classList.toggle），禁止 innerHTML 写入"。

### 5.4 规则冲突点 4：CSS hover

**现状**：`card.css` L1787 含 `.combined-tab:hover`。
**规则建议**：明确"所有 hover 效果禁止，但点击态（:active）和选中态（.active）允许"。
